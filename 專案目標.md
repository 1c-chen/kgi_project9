Project 9: The Manager Coaching Intervention Portal
The Business Context (The "Why")
When we transition from classroom training to self-paced micro-learning, we risk losing the "human touch" of traditional mentorship. If an agent fails a quiz on "Travel Insurance Data Sharing" three times in a row, they are likely frustrated and confused. If they keep guessing, they are reinforcing bad habits.
This system must know when to stop the digital learning process and trigger a human intervention. By temporarily locking the module and alerting the branch manager, we force a physical 1-on-1 coaching session. This ensures that the Financial Holding Company maintains high quality-control standards and that managers are actively involved in their team's professional development, rather than just passively looking at dashboards.
User Interface (UI)
●	The "Team Health" Roster: A clean, sorting-enabled list of the manager's direct reports. Agents are color-coded: Green (Healthy), Amber (Warning: Score dropping), and Red (Locked: Requires Intervention).
●	The Diagnostic View: When a manager clicks on a "Red" agent, they see exactly why the lock occurred. It displays the specific module, the questions the agent kept failing, and the AI's generated feedback (from Project 2).
●	The Intervention Log Form: A simple form where the manager logs the offline coaching session. It requires a text input (e.g., "Sat down with Lin; explained the FSC data privacy clause") and a digital signature button to "Unlock Agent's Module."
User Requirement (Spec)
1.	The Trigger Engine: The backend must evaluate quiz results in real-time. If an agent fails the same module 3 times, or their 7-day rolling average score drops below 70%, the system changes their state for that module to Locked_Pending_Coaching.
2.	The Notification Routing: The system must identify the agent's direct manager and send them an urgent notification (integrating with Project 8).
3.	The Unlock Protocol: The agent cannot progress in that specific learning path until the manager accesses the portal, fills out the mandatory coaching text field (enforcing a minimum character count so they don't just type "ok"), and hits submit. This action resets the agent's state to Active.
Database Design
The database must handle organizational hierarchy and maintain a strict audit trail of managerial actions.
1.	TeamStructures (The Org Chart)
○	mapping_id (Primary Key)
○	agent_id
○	manager_id
○	branch_code (e.g., 'TPE-Nangang')
○	effective_date (To handle transfers between branches)
2.	ModuleStateLocks (The Access Control)
○	lock_id (Primary Key)
○	agent_id
○	module_id
○	lock_reason (e.g., 'failed_3x')
○	locked_timestamp
○	is_locked (Boolean)
3.	CoachingInterventions (The Audit Trail)
○	intervention_id (Primary Key)
○	lock_id (Foreign Key)
○	manager_id
○	manager_notes_text
○	unlocked_timestamp
